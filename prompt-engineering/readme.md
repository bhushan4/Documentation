# ai-backend-engineering-guide

A structured, hands-on roadmap to becoming an 8/10 AI-powered backend engineer.

Built for engineers working with **Go** and **Java** who want to use AI tools (Claude, Copilot, Cursor, etc.) at a professional level — not just for autocomplete, but for architecture, code review, migrations, interviews, and stakeholder demos.

## What's Inside

**[AI-ENGINEERING-GUIDE.md](./AI-ENGINEERING-GUIDE.md)** — The full guide covering:

- **Phase 1** — Prompt foundations (RCTF framework, delimiters, few-shot, chain-of-thought)
- **Phase 2** — Starting new projects with AI (spec-first, harness engineering, context files)
- **Phase 3** — Legacy code with AI (review, refactor, migrate, chunk large codebases)
- **Phase 4** — Interview and live coding with AI (feature builds, code reviews, system design)
- **Phase 5** — Stakeholder demos (structure, metrics, handling objections)
- **Prompt Templates Library** — Copy-paste templates for daily use
- **Progress Tracker** — Track your journey from 0 → 8

## Quick Start

1. Read Phase 1 of the guide
2. Complete the practice exercises before moving on
3. Create `CLAUDE.md` and `CONVENTIONS.md` for your own repos (templates inside)
4. Track your progress with the built-in tracker at the bottom

## Time Estimate

4–6 weeks of deliberate practice (1–2 hours/day).

## Who This Is For

- Backend engineers (Go, Java) wanting to level up with AI
- Engineers preparing for interviews where AI tools are allowed
- Tech leads who need to demo AI workflows to stakeholders
- Teams adopting AI-assisted development practices
